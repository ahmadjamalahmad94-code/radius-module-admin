# نظام إدارة صفحة الهبوط (Landing Page CMS)

يتيح للمدير تعديل/إظهار/إخفاء/ترتيب وتحديث كل محتوى صفحة الهبوط العامة **من لوحة الإدارة بدون تعديل الكود**.

نموذج التشغيل: **حفظ فوري مباشر** — أي تعديل يُحفظ يظهر مباشرة في الصفحة العامة (مع حالة `draft/published/archived` للصفحة ككل).

---

## نظرة عامة
- الواجهة الإدارية: قسم **«إدارة الصفحة الرئيسية»** في القائمة الجانبية (`/admin/landing/`).
- العرض العام: `GET /` يحمّل صفحة `home` المنشورة ويرسم الأقسام الظاهرة مرتّبة، ولكل قسم عناصره الظاهرة.
- المحتوى ليس مكتوبًا في القالب (عدا fallback آمن لو لا توجد صفحة منشورة).

---

## نموذج البيانات (`app/models.py`)
| الجدول | الغرض |
|--------|------|
| `landing_pages` | الصفحة: slug، عنوان، حالة، SEO، is_homepage، published_at |
| `landing_sections` | أقسام الصفحة: section_key (فريد للصفحة)، section_type، عناوين/أزرار/أيقونة، sort_order، is_visible، settings_json |
| `landing_items` | عناصر داخل الأقسام: item_type، عنوان/وصف/قيمة/شارة/سعر/مزايا، status_badge، sort_order، is_visible |
| `landing_social_links` | روابط التواصل (platform, url, icon, visible, order) |
| `landing_contact_methods` | طرق الاتصال (type, label, value, url, visible, order) |
| `landing_revisions` | لقطات/تدقيق (متاح للتوسعة) |

ملاحظات تقنية:
- JSON يُخزَّن كـ `Text` + خصائص (`settings`/`features`) — تجنّبنا اسم `metadata` المحجوز في SQLAlchemy.
- الجداول تُنشأ تلقائيًا عبر `db.create_all()`، والمحتوى الافتراضي عبر `seed_landing_defaults()` (idempotent — لا يكرّر عند إعادة التشغيل).

أنواع الأقسام `section_type`: `hero, trust_chips, stats, cards_grid, modules, integrations, steps, cta, faq, footer`.
أنواع العناصر `item_type`: `feature, module, integration, metric, step, faq, trust_chip, offer, footer_link`.
حالات الشارة `status_badge`: `متاح / قيد التجهيز / حسب الخطة / قريبًا`.

## كيف تعدّل الصفحة الرئيسية
1. لوحة الإدارة ← **إدارة الصفحة الرئيسية**.
2. عدّل **إعدادات الصفحة وSEO** والحالة (مسودة/منشورة) ثم احفظ.
3. في جدول **الأقسام**: رتّب (▲▼)، أظهِر/أخفِ، أو افتح **تعديل** للنصوص/الأزرار، أو **العناصر** لإدارة البطاقات.

### إضافة بطاقة ميزة
العناصر ← إضافة عنصر ← النوع `feature` ← العنوان + الوصف + الأيقونة ← حفظ.

### إضافة سؤال شائع
داخل قسم FAQ ← إضافة عنصر ← النوع `faq` ← **العنوان = السؤال**، **الوصف = الإجابة**.

### إضافة عرض (offer)
النوع `offer` ← السعر/السعر القديم/المدة + **المزايا (ميزة في كل سطر)** + نص ورابط الزر.
> العروض حاليًا بطاقات تسويقية فقط وغير مربوطة بنظام الفوترة.

### روابط التواصل
**روابط التواصل** ← اختر المنصة + الرابط + ظاهر. لا تُعرض الروابط الفارغة أو المخفية. لواتساب استخدم `https://wa.me/الرقم`.

### إخفاء خدمة غير متاحة
بدل حذفها، اضبط `status_badge` على **«قريبًا»** أو **«قيد التجهيز»**، أو أخفِ العنصر.

## ملاحظات الأمان
- كل مسارات `/admin/landing/*` محمية بـ `@login_required` (مدير فقط).
- حماية CSRF مفعّلة على كل النماذج (`csrf_input()` + فحص `before_request`).
- الروابط تُفحَص (`_safe_url`) لرفض `javascript:`/`data:`.
- النصوص تُهرَّب تلقائيًا عند العرض (Jinja autoescape) — لا HTML/JS حر من المدير.
- كل تعديل يُسجَّل في `audit_logs` عبر `audit()`.
- لا تُعرض الروابط/طرق الاتصال الفارغة أو المخفية — ولا نخترع بيانات اتصال.

---

## تحسينات مستقبلية
- مسودة/نشر على مستوى القسم (حاليًا الحفظ فوري + حالة على مستوى الصفحة).
- استرجاع من `landing_revisions` (لقطات).
- رفع صور بدل روابط URL.
- صفحات هبوط متعددة (slugs) بنفس المحرّك.
